## 一、产品概述

### 1.1 产品定位

AI短剧协作平台是一个面向影视创作团队的多角色协作工作台，支持从剧本创作到分镜生成、视频制作的完整生产流程。平台采用导演-工作人员双层协作模式，导演负责项目管理与质量把控，工作人员负责具体创作执行。

### 1.2 核心价值

- **角色分工明确**：导演专注管理与审核，工作人员专注创作
- **流程标准化**：剧本→分镜→图片→视频→审核，步步可追踪
- **高效协作**：任务分配、进度可视、反馈即时
- **AI赋能**：文生图、图生视频等AI能力集成

### 1.3 目标用户

- **导演（Director）**：项目管理者，负责剧本上传、任务分配、进度监控、质量审核
- **工作人员（Staff）**：创作执行者，负责剧本拆解、分镜制作、内容生成

## 二、角色权限体系

### 2.1 导演角色权限

| 权限项 | 说明 |
|--------|------|
| 剧本管理 | 创建/上传剧本、查看剧本列表、管理剧本状态 |
| 任务分配 | 将剧本分配给指定工作人员 |
| 项目监控 | 查看所有项目进度、风险预警 |
| 质量审核 | 审核工作人员提交的成果、通过/驳回 |
| 团队管理 | 查看团队成员工作状态 |

### 2.2 工作人员角色权限

| 权限项 | 说明 |
|--------|------|
| 作品查看 | 查看分配给自己的剧本列表 |
| 剧本拆解 | 将剧本拆分为分镜卡片 |
| 分镜编辑 | 编辑分镜详情（角色、场景、镜头、情绪、台词、Prompt） |
| AI生成 | 触发文生图、图生视频等AI能力 |
| 任务提交 | 提交分镜成果供导演审核 |
| 迭代修改 | 根据审核反馈修改分镜内容 |

## 三、核心功能模块

### 3.1 用户认证模块

#### 3.1.1 登录页

**功能描述**：用户名密码登录，支持注册

**页面元素**：
- Logo与品牌标识
- 用户名输入框
- 密码输入框
- 角色选择（注册时）：导演/工作人员
- 登录/注册按钮
- 切换登录/注册模式链接

**交互逻辑**：
- 登录成功后根据角色跳转至对应工作台
- 失败时显示错误提示
- 注册时需选择角色类型

#### 3.1.2 身份状态管理

- 登录后保存用户信息（用户名、角色、Token）
- 未登录状态重定向至登录页
- 角色权限控制页面访问与操作按钮

### 3.2 导演工作台

#### 3.2.1 导演首页（仪表盘）

**功能描述**：展示导演关注的核心数据与待办事项

**页面布局**：

**顶部区域**：
- 欢迎语 + 快捷操作入口

**统计卡片区域**：
- 我的剧本总数
- 待处理任务数（工作人员提交的审核）
- 进行中项目数
- 团队成员数

**待审核列表**：
- 展示工作人员提交待审核的项目
- 字段：项目名、提交人、提交时间、当前状态
- 操作按钮：进入审核、查看详情

**项目进度概览**：
- 所有项目当前步骤统计

**快速入口**：
- 上传剧本
- 创建项目
- 查看团队

#### 3.2.2 剧本管理页

**(1) 剧本列表**

**功能描述**：展示所有已上传/创建的剧本，支持筛选、搜索、排序

**页面元素**：
- **搜索栏**：按剧本名称搜索
- **筛选器**：按状态筛选（草稿/已分配/进行中/审核中/已完成/已驳回）
- **排序选项**：按创建时间/更新时间/标题排序

**剧本卡片/列表字段**：
- 剧本标题
- 创建时间
- 当前状态标签
- 分配状态（未分配/已分配给XXX）
- 工作人员姓名（已分配时显示）
- 操作按钮：查看详情、分配任务、编辑、删除

**操作**：
- 新建剧本按钮：跳转到剧本创建页

**(2) 上传/创建剧本**

**功能描述**：导演上传新剧本或在线创建剧本

**页面元素**：
- 剧本标题输入框
- 剧本内容编辑区（富文本或多段落）
- 附件上传区（可选上传Word/PDF/TXT文件）
- 备注说明（可选）
- 操作按钮：保存草稿、发布并分配

**交互逻辑**：
- **保存草稿**：仅保存，不进入工作流
- **发布并分配**：保存后弹出分配对话框

**(3) 剧本详情页**

**功能描述**：查看剧本完整信息与关联项目进度

**页面内容**：
- 剧本基本信息（标题、创建时间、状态、分配信息）
- 剧本内容展示
- 关联项目列表（如有）
- 操作历史时间线（创建、分配、提交审核、审核结果等）

#### 3.2.3 任务分配功能

**(1) 分配对话框/页面**

**功能描述**：将剧本分配给指定工作人员

**页面元素**：
- 剧本标题展示
- 工作人员选择器（下拉选择或搜索选择）
  - 显示工作人员列表（姓名、当前任务数、空闲状态）
- 任务说明/要求输入框（可选）
- 截止日期选择器（可选）
- 操作按钮：确认分配、取消

**交互逻辑**：
- 选择工作人员后可看到其当前工作负载
- 分配成功后发送通知（站内或邮件）
- 剧本状态变更为"已分配"

**(2) 已分配任务管理**

- 查看已分配出去的剧本列表
- 查看每个任务的执行进度
- 支持重新分配、撤销分配（仅限未开始的任务）

#### 3.2.4 审核管理页

**(1) 待审核列表**

**功能描述**：展示所有待审核的项目

**列表字段**：
- 项目名称
- 提交工作人员
- 提交时间
- 分镜数量
- 当前状态
- 操作：进入审核、查看详情

**(2) 审核会话页**

**功能描述**：审核工作人员提交的分镜成果

**页面布局**：

**左侧 - 项目基本信息**：
- 项目名、工作人员、提交时间
- 分镜总数、通过数、待审数

**中部 - 分镜预览区**：
- 分镜卡片轮播或网格展示
- 每个分镜显示：序号、场景描述、生成的图片/视频
- 点击可放大查看

**右侧 - 审核操作区**：
- 审核意见输入框
- 批量操作：全部通过、全部驳回
- 单个分镜操作：通过/驳回
- 驳回原因必填

**底部 - 审核历史**：
- 历次审核记录
- 审核人、时间、结果、意见

**交互逻辑**：
- **通过**：项目状态变更为"已通过"，进入导出阶段
- **驳回**：项目状态变更为"已驳回"，回流至工作人员修改
- 驳回时必须填写原因，工作人员可见

#### 3.2.5 项目监控页

**(1) 项目列表**

- 所有项目概览
- 筛选：按状态、工作人员、时间
- 排序：按进度、创建时间

**项目卡片显示**：
- 项目名称
- 当前步骤（剧本/分镜/生成/审核）
- 负责人
- 进度百分比
- 状态标签
- 快捷操作：查看详情、催办

**(2) 项目详情页（驾驶舱）**

- **顶部**：步骤条展示当前进度
- **中部三栏**：
  - 项目信息（标题、负责人、创建时间）
  - 风险提醒（阻塞项、失败重试）
  - 资产统计（分镜数、图片数、视频数）
- **底部**：流程时间线
  - 所有操作记录按时间倒序
  - 包括：创建、分配、提交、审核、驳回等

### 3.3 工作人员工作台

#### 3.3.1 工作人员首页（工作台）

**功能描述**：展示工作人员的工作任务与进度

**页面布局**：

**顶部 - 欢迎语 + 个人统计**：
- 我的作品总数
- 进行中任务数
- 待修改数（被驳回的）
- 已完成数

**任务看板**：
- 待处理：新分配的剧本
- 进行中：正在编辑/生成的项目
- 待审核：已提交等待导演审核
- 需修改：被驳回需要修改的

**其他**：
- 快速入口：进入作品、开始创作
- 最近活动：最近的操作记录

#### 3.3.2 作品列表页

**(1) 我的作品列表**

**功能描述**：展示所有分配给当前工作人员的剧本/项目

**页面元素**：

**筛选器**：
- 按状态：待处理/进行中/已提交/已通过/已驳回
- 按时间：最近分配/最近更新

**作品卡片**：
- 作品标题（剧本名）
- 分配时间
- 分配人（导演姓名）
- 当前状态标签
- 进度指示（分镜完成数/总数）
- 操作按钮：继续编辑、查看详情、提交审核

**空状态**：暂无分配的作品

**(2) 作品详情页**

- 作品基本信息
- 剧本内容查看
- 分镜列表与编辑入口
- 提交审核按钮
- 审核反馈查看（如被驳回）

#### 3.3.3 剧本拆解与分镜工作台

**(1) 分镜工作台（核心页面）**

**功能描述**：工作人员拆解剧本、编辑分镜、生成内容的主工作区

**页面布局（三栏式）**：

**左侧 - 分镜卡片流**：
- 分镜列表（可滚动）
- 每个分镜卡片显示：
  - 分镜序号
  - 简要描述
  - 状态指示（待编辑/图片就绪/视频就绪）

**操作按钮**：
- 新建分镜
- 批量删除
- 排序/拖动排序（可选）

**中部 - 分镜预览区**：
- 当前选中分镜的预览
- 显示生成的图片/视频
- 16:9比例展示
- 无内容时显示占位符提示

**图片/视频生成按钮**：
- 文生图按钮
- 图生视频按钮
- 加载状态指示器

**右侧 - 编辑面板**：
- 分镜序号输入
- 角色输入（逗号分隔）
- 场景描述文本区
- 台词文本区
- 镜头语言输入（如：特写、远景、俯拍等）
- 情绪输入（如：紧张、温馨、悬疑等）
- Prompt文本区（AI生成用提示词）

**操作按钮**：
- 保存分镜（绿色主按钮）
- 提交审核（金色次要按钮）

**底部 - 评论/反馈区（可选）**：
- 导演审核意见展示
- 备注编辑区

**(2) 分镜状态流转**

```
待编辑 → 编辑中 → 图片生成中 → 图片就绪 → 视频生成中 → 视频就绪 → 提交审核
```

**(3) AI生成功能**

**文生图**：
- 根据Prompt调用AI生成图片
- 显示生成进度
- 生成成功后展示图片
- 支持重新生成

**图生视频**：
- 基于已生成的图片生成视频
- 显示生成进度
- 生成成功后展示视频
- 支持重新生成

**(4) 批量操作**

- 批量生成图片
- 批量生成视频
- 批量提交审核
- 批量重试失败任务

#### 3.3.4 生成队列页

**功能描述**：查看AI生成任务的状态

**页面元素**：

**任务列表**：
- 分镜序号
- 任务类型（图片/视频）
- 状态（排队中/生成中/成功/失败）
- 进度条
- 失败原因（如失败）

**操作**：
- 刷新状态
- 重新生成（失败时）
- 批量重试
- 自动轮询更新状态（每5秒）

#### 3.3.5 提交审核

**功能描述**：工作人员完成分镜后提交给导演审核

**触发条件**：
- 至少有一个分镜
- 所有分镜已完成编辑
- 关键分镜已有图片/视频

**交互流程**：
1. 点击"提交审核"按钮
2. 弹出确认对话框
3. 填写提交说明（可选）
4. 确认提交
5. 项目状态变更为"审核中"
6. 通知导演

### 3.4 通用功能

#### 3.4.1 顶部导航栏

**左侧**：Logo + 平台名称

**中部 - 导航标签**：
- 工作台（首页）
- 剧本/作品（根据角色显示不同文案）
- 审核（仅导演）
- 项目监控（仅导演）

**右侧**：
- 用户头像 + 姓名 + 角色
- 退出登录按钮

#### 3.4.2 页面标题栏

- 页面标题（大字号）
- 页面副标题（说明性文字）
- 面包屑导航（可选）

#### 3.4.3 步骤条组件

- 展示项目当前所处阶段
- 阶段：剧本 → 分镜 → 生成 → 审核 → 导出
- 当前步骤高亮
- 已完成步骤打勾
- 未完成步骤灰色

#### 3.4.4 状态标签系统

**剧本状态**：草稿、已分配、进行中、审核中、已通过、已驳回  
**分镜状态**：待编辑、编辑中、图片就绪、视频就绪、已审核  
**任务状态**：排队中、生成中、成功、失败  

**标签样式**：不同颜色区分（金/蓝/绿/黄/红/灰）

#### 3.4.5 时间线组件

- 垂直时间线展示操作历史
- 每个节点包含：操作类型、操作人、时间、备注
- 节点类型用不同图标/颜色区分

#### 3.4.6 Toast通知系统

- 操作成功/失败/警告提示
- 右上角弹出
- 3秒自动消失
- 支持手动关闭

#### 3.4.7 模态对话框

- 确认操作对话框
- 表单输入对话框
- 详情查看对话框
- 毛玻璃背景遮罩

## 五、数据模型与状态机

### 5.1 剧本（Script）数据模型

```json
{
  "id": "number",
  "title": "string - 剧本标题",
  "content": "string - 剧本内容",
  "status": "string - 草稿/已分配/进行中/审核中/已通过/已驳回",
  "director_id": "number - 创建导演",
  "assigned_to": "number - 分配的工作人员ID",
  "assigned_at": "datetime - 分配时间",
  "created_at": "datetime",
  "updated_at": "datetime"
}
```

### 5.2 项目（Project）数据模型

```json
{
  "id": "number",
  "title": "string - 项目名称",
  "script_id": "number - 关联剧本",
  "status": "string - 项目状态",
  "current_step": "string - 当前步骤",
  "director_id": "number - 负责导演",
  "staff_id": "number - 执行工作人员",
  "created_at": "datetime",
  "updated_at": "datetime"
}
```

### 5.3 分镜（Scene）数据模型

```json
{
  "id": "number",
  "project_id": "number",
  "scene_index": "number - 分镜序号",
  "characters": "string[] - 角色列表",
  "scene_description": "string - 场景描述",
  "dialogue": "string - 台词",
  "camera_angle": "string - 镜头语言",
  "emotion": "string - 情绪",
  "prompt": "string - AI提示词",
  "image_url": "string - 生成的图片URL",
  "video_url": "string - 生成的视频URL",
  "status": "string - 待编辑/编辑中/图片就绪/视频就绪/已审核",
  "created_at": "datetime",
  "updated_at": "datetime"
}
```

### 5.4 审核（Review）数据模型

```json
{
  "id": "number",
  "project_id": "number",
  "reviewer_id": "number - 审核人（导演）",
  "status": "string - approved/rejected",
  "comment": "string - 审核意见",
  "created_at": "datetime"
}
```

### 5.5 项目状态机

```
draft（草稿）
  → assigned（已分配）[导演分配]
  → storyboard_in_progress（分镜处理中）[工作人员开始编辑]
  → render_in_progress（生成处理中）[AI生成图片/视频]
  → in_review（审核中）[工作人员提交审核]
  → approved（通过）[导演审核通过]
  → rejected（驳回）[导演审核驳回]
    → storyboard_in_progress [工作人员修改后重新提交]
```

### 5.6 分镜状态机

```
todo（待编辑）
  → editing（编辑中）
  → image_ready（图片就绪）
  → video_ready（视频就绪）
  → accepted（已审核通过）
```

## 六、关键交互流程

### 6.1 剧本创建与分配流程

```
1. 导演登录 → 进入导演工作台
2. 点击"上传剧本"或"创建剧本"
3. 填写剧本信息 → 保存
4. 弹出分配对话框
5. 选择工作人员 → 确认分配
6. 剧本状态变更为"已分配"
7. 工作人员收到通知
```

### 6.2 工作人员执行流程

```
1. 工作人员登录 → 进入工作台
2. 查看"待处理"任务
3. 点击进入作品详情
4. 进入分镜工作台
5. 拆解剧本：创建分镜卡片
6. 编辑每个分镜：填写信息
7. 触发AI生成：文生图 → 图生视频
8. 完成所有分镜 → 点击"提交审核"
9. 项目状态变更为"审核中"
```

### 6.3 导演审核流程

```
1. 导演收到审核通知
2. 进入审核列表 → 点击进入审核
3. 查看分镜预览（图片/视频）
4. 逐个审核或通过批量操作
5. 通过：项目进入"已通过"状态
6. 驳回：填写原因 → 项目回流至工作人员
7. 工作人员修改后重新提交
```

## 七、UI/UX设计规范

### 7.1 视觉风格

- **主题**：电影工业风、暗色主题
- **主色**：黑色背景 `#0a0a0f`
- **强调色**：金色渐变 `#ffd700 → #ffb700`
- **辅助色**：
  - 蓝色：操作按钮、链接
  - 绿色：成功状态
  - 红色：警告/错误/驳回
  - 紫色：AI生成相关
- **卡片样式**：毛玻璃效果、金色边框高光
- **按钮样式**：渐变色、阴影、hover动效

### 7.2 布局规范

- **顶部导航**：64px高度、固定定位、毛玻璃背景
- **页面标题栏**：32px上下内边距、白色大标题
- **内容区**：32px上下左右内边距
- **卡片间距**：16px（小）/ 24px（中）
- **栅格系统**：12列栅格、响应式布局

### 7.3 交互规范

- **按钮反馈**：hover变色 + 阴影加深 + 微上浮
- **卡片反馈**：hover边框发光 + 阴影 + 微上浮
- **页面切换**：fade-in动画（0.5s）
- **加载状态**：旋转指示器 + 文字提示
- **空状态**：插画 + 说明文字 + 操作引导

### 7.4 响应式设计

- **桌面端（≥1280px）**：完整三栏/多栏布局
- **平板端（768px-1279px）**：两栏布局
- **移动端（<768px）**：单栏堆叠、汉堡菜单（可选）

## 八、非功能性需求

### 8.1 性能要求

- 页面加载时间 < 2秒
- API响应时间 < 500ms
- 图片/视频生成异步处理，不阻塞界面
- 列表分页加载，避免一次性加载过多数据

### 8.2 可用性要求

- 99.9%可用性
- 错误处理：网络错误、服务器错误、权限错误均有提示
- 操作撤销：关键操作提供撤销/确认机制
- 数据保存：编辑中自动保存草稿

### 8.3 安全性要求

- JWT Token认证
- 角色权限校验（前端+后端双重）
- XSS/CSRF防护
- 敏感操作二次确认
